🚨 PROBLEMAS IDENTIFICADOS
─────────────────────────────────────────────────────────────────────────────

CRÍTICOS (Afetam UX Imediatamente):

1. SEM BREAKPOINT 375px
   • 30-40% dos usuários mobile não têm layout otimizado
   • Usa regras de 480px (muito grande)
   • Impacto: Teste com iPhone SE - fonte gigante, botões inacessíveis

2. BOTÕES INACESSÍVEIS
   • .lang-btn com 9px em 768px
   • Altura efetiva: ~20px (metade do recomendado)
   • WCAG recomenda: 44-48px
   • Impacto: 15-20% população tem dificuldade motora

3. GRID PROCESSO - 4 COLUNAS EM 1200px
   • 1200px: grid com 4 colunas em espaço de 2
   • Texto ilegível em 1200px-1366px
   • Só reduz para 2 em 768px
   • Impacto: 25% do tráfego desktop quebrado

4. PADDING EXCESSIVO
   • Desktop: padding-left: 96px
   • Salta para 40px em 1200px
   • Depois para 28px em 768px
   • Resultado: Layout shift, desconforto visual


GRAVES (Afetar Experiência):

1. TRANSIÇÕES ABRUPTAS
   • Redução de padding: 64px → 28px (56% em 1200px → 768px)
   • Não há breakpoint intermediário em 900px
   • Penalizado por Google (CLS - Cumulative Layout Shift)

2. ALTURAS DE IMAGENS FIXAS
   • Desktop: 100% (full height)
   • 1200px: 520px (fixo)
   • 768px: 380px (fixo)
   • Falta: 900px, 600px, 375px

3. GRID SEM TRANSIÇÃO
   • Diferenciais: múltiplas colunas → 2 colunas (1200px) → 1 coluna (768px)
   • Sem breakpoint em 600px para transição suave

4. ELEMENTO VISUAL REMOVIDO
   • .c-eye { display: none; } em 480px
   • Logo fica incompleto visualmente
   • Afeta branding em 30% dos casos


⚠️ AVISOS (Verificar):

1. Font-size de headings em 375px (pode ser pequena)
2. Botões podem ser muito grandes em 375px (sem media query)
3. Possível horizontal scroll em alguns elementos
