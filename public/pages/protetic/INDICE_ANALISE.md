## 🔑 Problemas Chave (Por Urgência)

### 🚨 CRÍTICOS (Implementar AGORA)
1. **Sem Breakpoint 375px** - 30-40% dos usuários mobile
   - Afeta: iPhone SE, 11, 12 mini, Galaxy S10e
   - Solução: Ver GUIA_IMPLEMENTACAO.md
   - Código: melhorias_responsivas.css (linhas ~360-420)

2. **Botões Inacessíveis (9px)** - 50% dos usuários
   - Afeta: Language selectors, CTAs
   - Solução: Min-height 44-48px
   - Código: melhorias_responsivas.css (linhas ~200-220)

3. **Processo 4 Colunas em 1200px** - 25% do tráfego
   - Afeta: Seção de processo/timeline
   - Solução: Mudar para 2 colunas em 1200px
   - Código: melhorias_responsivas.css (linha ~120)

4. **Padding Excessivo** - Todos os usuários
   - Afeta: Navegação, layout geral
   - Solução: Usar variáveis CSS
   - Código: melhorias_responsivas.css (linhas ~30-50)

## 🎓 Apêndice: Termos-Chave

**Breakpoint**: Largura de viewport onde CSS muda (ex: 768px)

**Media Query**: Regra CSS que aplica estilos baseado em condições (ex: @media (max-width: 768px))

**Responsivo**: Site adapta-se a diferentes tamanhos de tela

**WCAG**: Diretrizes de acessibilidade web

**CLS (Cumulative Layout Shift)**: Métrica que mede quanto o layout se move (ruim se >0.1)

**Hit Area**: Área clicável de um botão (mín. 44x44px recomendado)

**Viewport**: Área visível da página no navegador
