🚨 Problemas Críticos

1. Sem Breakpoint para 375px (iPhone SE/11)

O site usa regras de 480px para tudo abaixo disso, deixando iPhones padrão com layout possivelmente apertado.

2. Botões de Language Inacessíveis

.lang-btn com font-size: 9px em 768px é extremamente pequeno e difícil de clicar.

  • Altura mínima: ~20px
  • Recomendado: mínimo 44-48px
  • Afeta: Seletores de idioma PT/EN/ES
3. Grid de Processo com 4 Colunas em 1200px

O breakpoint 1200px mantém 4 colunas na timeline do processo, muito apertado para a resolução.

.proc-tl { grid-template-columns: repeat(4,1fr) !important; } /* em 1200px */
4. Padding Excessivo em Desktop

Navigation com padding: 96px deixa muito espaço vazio em screens > 1920px.

⚠️ Problemas Graves

1. Transição Abrupta de Padding

1200px: 64px → 768px: 28px (redução de 56%!)

Pode causar layout shift desconfortável ao redimensionar a janela.

2. Altura de Imagens Não Responsiva

Imagens têm altura fixa em breakpoints, sem transição suave:

  • Desktop: 100% height
  • 1200px: 520px
  • 768px: 380px
  • Falta: Breakpoint em 600px e 375px
3. Grids sem Transição Intermediária

Diferenciais: Desktop (múltiplas colunas) → 1200px (2 colunas) → 768px (1 coluna)

Sem breakpoint em 600px para transição suave.

4. Elemento Visual Removido em 480px

.c-eye { display: none; } - Logo perde elemento visual importante.

💡 Avisos (Verificar Implementação)

1. Font-Size de Headings em Mobile

Heading .H reduz de 38px (768px) para 32px (480px).

Em 375px, será 32px - verificar legibilidade.

2. Botões Potencialmente Grandes em 375px

.btn-p sem media query para 375px pode ficar muito grande ou com overflow.

3. Possível Horizontal Scroll

Conteúdo com overflow-x: hidden pode estar ultrapassando viewport em resoluções não cobertas.

✅ Recomendações de CSS Responsivo

1. Adicionar Breakpoint para 375px

@media (max-width: 375px) {
  .c-mark { font-size: 48px !important; }
  .H { font-size: 28px !important; }
  .con-ttl { font-size: 32px !important; }
  section { padding-left: 16px; padding-right: 16px; }
  nav { padding: 14px 16px !important; }
  .lang-btn { padding: 5px 8px !important; font-size: 8px !important; }
  .id-content, .frase-content, .tech-content, .eq-content, .val-content {
    padding: 32px 16px !important;
  }
}

2. Adicionar Breakpoint Intermediário 600px

@media (max-width: 600px) {
  .diff-grid { grid-template-columns: 1fr !important; }
  .proc-tl { grid-template-columns: 1fr !important; gap: 24px; }
  nav { padding: 14px 20px !important; }
  .H { font-size: 34px !important; }
}

3. Corrigir Grid de Processo em 1200px

@media (max-width: 1200px) {
  .proc-tl { grid-template-columns: repeat(2, 1fr) !important; }
  /* Mudar de 4 para 2 colunas */
}

4. Usar Variáveis CSS para Padding

:root { --p-h: 96px; --p-v: 80px; }
@media (max-width: 1200px) { --p-h: 64px; --p-v: 72px; }
@media (max-width: 900px) { --p-h: 48px; --p-v: 56px; }
@media (max-width: 768px) { --p-h: 40px; --p-v: 52px; }
@media (max-width: 480px) { --p-h: 28px; --p-v: 40px; }
@media (max-width: 375px) { --p-h: 20px; --p-v: 32px; }

section { padding: var(--p-v) var(--p-h); }

5. Melhorar Acessibilidade de Botões

.lang-btn {
  min-height: 44px;
  min-width: 44px;
  padding: 8px 16px;
}

.btn-p {
  min-height: 48px;
  padding: 12px 24px;
}

@media (max-width: 768px) {
  .lang-btn { min-height: 40px; padding: 8px 12px; }
  .btn-p { min-height: 44px; padding: 12px 20px; }
}

6. Transição Suave de Alturas de Imagens

.id-img, .frase-img, .tech-img, .eq-img, .val-img {
  height: 100%;
}
@media (max-width: 1200px) { height: 520px; }
@media (max-width: 900px) { height: 450px; }
@media (max-width: 768px) { height: 380px; }
@media (max-width: 600px) { height: 340px; }
@media (max-width: 480px) { height: 280px; }
@media (max-width: 375px) { height: 240px; }

🎯 Conclusão Final

O site Protetic possui uma estrutura CSS responsiva bem organizada com media queries em pontos estratégicos. No entanto, apresenta 4 problemas críticos e 4 problemas graves que afetam a experiência em dispositivos mobile, especialmente em:

  • iPhone SE/11 (375px) - Sem breakpoint específico
  • Botões inacessíveis - Muito pequenos em mobile
  • Grid de processo - 4 colunas em 1200px é inacessível
  • Transições abruptas - Mudanças de padding muito drásticas

Implementar as recomendações acima aumentará significativamente a acessibilidade e experiência do usuário em todos os dispositivos, especialmente em mobile.
