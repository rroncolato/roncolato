🧪 Guia de Testes Práticos - Protetic

Manual interativo para testar a responsividade do site em diferentes resoluções

1. Testes em Desktop (1920px)

💻 Desktop Padrão - 1920x1080

Visual & Layout

Logo/Marca visível e completa

Verificar: .c-eye deve estar visível, marca com tamanho apropriado (~100px)

Navegação com padding 96px

Abrir DevTools (F12) → Inspecionar nav → Verificar padding: 28px 96px

Seções em layout 50%-50%

Identidade, Frase, Equipe devem ter 2 colunas lado a lado

Imagens com altura 100% (full viewport)

Scroll suave, sem cortes de imagem

Processo com 4 colunas

Timeline deve mostrar 4 passos lado a lado

Nenhum elemento decorativo escondido

Elementos como .c-eye, .frase-bg-txt devem estar visíveis

Performance & Animações

Scroll suave funcionando

Verificar: scroll-behavior: smooth; está funcionando

Navigation transition ao scroll

Rolar página: nav deve ter background escuro e padding reduzido

Sem CLS (Cumulative Layout Shift)

Layout não deve pular ao carregar. F12 → Lighthouse para verificar

2. Testes em Tablet Grande (1200px)

📱 Tablet/Laptop Pequeno - 1200x800
⚠️ Este é um breakpoint problemático!
Verificar se as correções propostas funcionam corretamente.

Problemas a Verificar (ANTES da implementação)

⚠️ PROBLEMA: Processo com 4 colunas MUITO APERTADO

Resultado esperado: Texto quebrado, ilegível, ~280px por coluna

⚠️ PROBLEMA: Layout shift ao redimensionar

Redimensionar janela 1920px → 1200px: layout deve pular

⚠️ PROBLEMA: Padding reduzido abruptamente

Padding muda de ~96px (1920px) para 40px (1200px) = 56% de redução

✅ Após implementar melhorias:
• Processo com 2 colunas (não 4)
• Padding suave: 96px → 64px → 48px (transição em 900px)
• Layout stable, sem shifts abruptos

Testes Pós-Implementação

Processo com 2 colunas (NÃO 4)

Deve alternar para 2 colunas em 1200px

Imagens com 520px altura

Proporção de imagem vs texto deve ser balanceada

Diferenciais com 2 colunas

Grid deve mostrar 2 itens por linha

3. Testes em Tablet (768px)

📱 Tablet Médio - 768x1024 (iPad)

Layout & Responsividade

Navigation com padding 40px horizontal

DevTools: F12 → nav → padding: 16px 24px (ou 40px conforme media query)

Imagens com 380px altura

Proporção equilibrada entre imagem e texto

Diferenciais em 1 coluna

Grid deve alternar para 1 item por linha

Processo em 2 colunas

Timeline com 2 passos por linha

Fontes legíveis (.H = 38px)

Headings devem ser fáceis de ler sem zoom

Acessibilidade

Botões de language com altura mínima

ANTES: 9px (muito pequeno) | DEPOIS: 40px mínimo

CTA buttons clicáveis

Mínimo 44-48px de altura

4. Testes em Mobile Médio (600px)

📱 Smartphone Médio - 600x800
⚠️ ATENÇÃO: Este breakpoint NÃO existe no CSS original!
Será adicionado na implementação de melhorias.

Testes Pós-Implementação

Imagens com 340px altura

Gradação: 380px (768px) → 340px (600px) → 280px (480px)

Processo em 1 coluna

Timeline empilhada verticalmente

Padding 32px horizontal

Transição suave entre 768px (40px) e 480px (24px)

Fontes reduzidas mas legíveis

.H = 34px, bom compromisso entre legibilidade e espaço

5. Testes em Mobile Padrão (480px)

📱 Smartphone Padrão - 480x812 (Galaxy S10)

Layout & Espaçamento

Sem horizontal scroll

Conteúdo deve caber em 480px sem overflow

Imagens com 280px altura

Adequado para mobile, deixa espaço para texto

Padding 24px horizontal

DevTools: F12 → Inspecionar section → padding: 36px 24px

Fontes ajustadas

.H = 32px, .c-mark = 52px

Processo em 1 coluna

Timeline empilhada, cada passo em uma linha

Logo com elemento visual (c-eye)

ANTES: display:none | DEPOIS: visível (reduzido)

Acessibilidade

Botões com altura mínima 44px

Toque confortável sem erros

Seletores de idioma clicáveis

ANTES: 9px (inacessível) | DEPOIS: 40px mínimo

Sem necessidade de zoom

Texto base 14px+, heading 28px+

6. Testes em iPhone Pequeno (375px) - ⚠️ CRÍTICO

📱 iPhone SE/11 - 375x812 (NOVO BREAKPOINT)
🚨 CRÍTICO: Este é o breakpoint FALTANTE!
Será adicionado na implementação.
ANTES: Usa regras de 480px (muito grande)
DEPOIS: Regras específicas para 375px

Problemas Atuais (ANTES da implementação)

⚠️ PROBLEMA: Fontes muito grandes

.c-mark = 52px (usa regra de 480px) → quebra em múltiplas linhas

⚠️ PROBLEMA: Imagens muito altas

280px em viewport de 375px = 75% da altura

⚠️ PROBLEMA: Botões inacessíveis

Language buttons muito pequenos, difícil clicar

⚠️ PROBLEMA: Possível horizontal scroll

Conteúdo pode ultrapassar 335px (375 - 40px padding)

✅ Após implementar breakpoint 375px:
• .c-mark reduzido a 48px
• .H reduzido a 28px
• Imagens reduzidas a 240px
• Padding 20px horizontal
• Botões com altura mínima 40px
• Sem horizontal scroll

Testes Pós-Implementação

Logo com fonte 48px (reduzida de 52px)

Deve quebrar apenas 1 linha, não 2-3

Imagens com 240px altura

Proporção melhor: 64% da altura em vez de 75%

Padding reduzido a 20px horizontal

Espaço disponível: 335px (muito melhor que ~240px)

Botões com 40px mínimo de altura

WCAG compliant, fácil de clicar com dedos

Sem horizontal scroll

Todos elementos dentro do viewport

Logo com elemento visual visível

.c-eye mantém com tamanho reduzido (~70px)

Processo em 1 coluna

Timeline vertical, fácil de navegar

Sem necessidade de zoom

Texto base legível, heading 28px mínimo
