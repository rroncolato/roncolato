```bash
  --typescript \

  --eslint

# 4. Rodar development server
npm run dev

# 5. Build production
npm run build
npm start

# 6. Deploy Vercel
npm i -g vercel
vercel deploy

# 7. Run tests
npm test

# 8. Lint code
npm run lint
```

## 🎯 DECISION TREES

### "Qual framework escolher?"

```
Precisa de máxima performance?
  ├─ SIM → Next.js 15 (App Router) ✅
  └─ NÃO → HTML5 puro com JS vanilla

Vai usar React?
  ├─ SIM → Next.js ✅
  └─ NÃO → Astro ou HTML puro

Precisa de styling?
  ├─ Tailwind (utility-first) ✅ RECOMENDADO
  ├─ CSS-in-JS (Emotion)
  └─ CSS puro
```

### "Qual CSS framework?"

```
Precisa de componentes prontos?
  ├─ Bootstrap (174k⭐) ✅ Mais popular
  ├─ Foundation (acessibilidade)
  ├─ Pico CSS (minimalista)
  └─ Tailwind (utility-first)

Quer escrever menos CSS?
  ├─ Tailwind + HeadlessUI ✅
  ├─ Bootstrap
  └─ Foundation

Acessibilidade é prioridade?
  ├─ Foundation ✅ Accessibility-first
  ├─ Bootstrap
  └─ Material Design
```

### "Qual TypeScript?"

```
Novo em TypeScript?
  ├─ Comece com JSDoc + // @ts-check
  └─ Depois migre para .ts

Projeto novo?
  ├─ Sempre use TypeScript ✅
  └─ Benefícios: type safety, autocomplete

Framework decide?
  ├─ Next.js → TypeScript first
  ├─ React → TypeScript recommended
  └─ Vanilla JS → TypeScript optional
```

## 📝 NOTAS IMPORTANTES

### Não Faça

❌ HTML sem semantic tags (`<div>` everywhere)  
❌ Hardcoded colors (use CSS variables)  
❌ Animations in layout properties (animate only opacity/transform)  
❌ Images without alt text  
❌ Inaccessible forms (labels required)  
❌ Ignore Core Web Vitals  
❌ Desktop-first approach (mobile-first obrigatório)  
❌ Heavy JavaScript (lazy load tudo)  

### Sempre Faça

✅ Use semantic HTML (`<header>`, `<nav>`, `<main>`, etc)  
✅ Mobile-first CSS  
✅ Design tokens (CSS variables)  
✅ GPU-accelerated animations (opacity, transform)  
✅ Alt text on images  
✅ Keyboard accessible forms  
✅ Test on real mobile devices  
✅ Monitor Core Web Vitals  
✅ Optimize images (WebP, responsive, lazy)

## 📞 FAQ

### P: Qual é melhor, Next.js ou HTML puro?
**R:** Next.js para máxima performance e features. HTML puro para simplicidade. Para Roncolato, recomendamos Next.js 15.

### P: Preciso de Bootstrap ou Tailwind?
**R:** Tailwind é mais flexível e moderno. Bootstrap é mais pronto. Recomendamos Tailwind + componentes customizados.

### P: Quanto tempo leva implementar?
**R:** 4-6 semanas com 1 developer (tempo estimado):
- Semana 1-2: Setup + estrutura base
- Semana 3-4: Componentes principais
- Semana 5: Polishing + optimization
- Semana 6: Testing + deployment

### P: Qual é o custo?
**R:** 
- Hosting Vercel: $20/mês
- Domain: $12/ano
- Development: varies
- Zero custo de frameworks (open source)

### P: Como medir sucesso?
**R:** Métricas chave:
- Core Web Vitals (LCP, INP, CLS)
- Lighthouse score (90+)
- Conversão (CTA clicks → appointments)
- Mobile traffic
- User engagement (time on site, bounce rate)

### P: Preciso de designer?
**R:** Recomendado para:
- Logo refinement
- Photo shoots
- Brand guidelines
- Figma design system

### P: Como fazer manutenção pós-launch?
**R:** 
- Monitorar Core Web Vitals 2x/semana
- Check Analytics para user behavior
- Atualizações de conteúdo (dentistas, serviços)
- Security patches (Next.js + dependencies)
- A/B testing CTAs
