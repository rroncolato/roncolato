### A. DESIGN PATTERNS MODERNOS

| Padrão | Descrição | Quando Usar | Implementação |
|--------|-----------|------------|----------------|
| **Mobile-First Design** | Começar com design para mobile (320px), expandindo progressivamente | Sempre (padrão 2025) | Base CSS otimizada para mobile, media queries para tablets/desktop |
| **Semantic HTML Structure** | Uso correto de `<header>`, `<nav>`, `<main>`, `<article>`, `<footer>` | Sempre - essencial para acessibilidade e SEO | Estrutura lógica sem `<div>` quando há alternativa semântica |
| **Glassmorphism** | Efeito de vidro fosco com blur e transparência | Para designs modernos premium, CTAs especiais | `backdrop-filter: blur()`, RGBA colors, layered backgrounds |
| **Micro-interactions** | Pequenas animações em resposta a ações do usuário | Feedback visual, indicadores de estado, hover effects | CSS transitions em opacity/transform, JavaScript event handlers |
| **Component-Based Design** | Descomposição em componentes reutilizáveis | Todos os projetos modernos (escalabilidade) | Design systems com componentes reusáveis documentados |

### D. ACCESSIBILITY STANDARDS (WCAG 2.2)

| Padrão | Implementação | Benefício |
|--------|--------------|-----------|
| **Semantic HTML** | `<header>`, `<nav>`, `<main>`, `<article>`, `<footer>`, proper heading hierarchy | Screen readers entendem estrutura, SEO melhorado |
| **Alt Text para Imagens** | `alt="descrição precisa"`, incluir terapia se relevante | Usuários cegos/low vision entendem conteúdo |
| **Keyboard Navigation** | Todos elementos clicáveis acessíveis com Tab + Enter, focus visible | Usuários sem mouse conseguem navegar |
| **Color Contrast** | Mínimo 4.5:1 para texto, 3:1 para graphics (WCAG AA) | Usuários com baixa visão conseguem ler |
| **Form Labels** | `<label>` associadas aos inputs via `for` attribute | Screen readers conectam label ao input |
| **ARIA Attributes** | `aria-label`, `aria-describedby`, `role`, quando semantic não suficiente | Melhor semântica para assistive tech |
| **Focus Management** | Visible focus states (outline), skip links, logical tab order | Keyboard users veem onde estão |

### E. RESPONSIVE DESIGN APPROACHES

| Abordagem | Implementação | Detalhes |
|-----------|--------------|----------|
| **Mobile-First CSS** | Base styles para mobile, media queries para tablet/desktop | `min-width: 768px`, `min-width: 1024px`, `min-width: 1440px` |
| **Flexible Grid** | CSS Grid 2D + Flexbox 1D, nunca fixed widths | `grid-template-columns: repeat(auto-fit, minmax(300px, 1fr))` |
| **Fluid Typography** | Font sizes que escalam com viewport | `font-size: clamp(1rem, 2vw, 2rem)` technique |
| **Responsive Images** | `<picture>`, `srcset`, `sizes` attributes | Diferentes imagens para diferentes dispositivos |
| **Breakpoint Strategy** | 3-4 breakpoints máximo (320px, 768px, 1024px, 1440px) | Avoid device-specific widths, focus on layout breaks |
| **Touch-Friendly Targets** | Botões/links com mínimo 44x44px de toque area | Móvel-first consideration |

### F. ANIMATION & INTERACTION PATTERNS

| Padrão | Implementação | Performance | Uso |
|--------|--------------|-------------|-----|
| **Smooth Transitions** | `transition: opacity 0.3s ease-in-out;` | GPU-accelerated (opacity, transform) | Hover states, state changes |
| **Keyframe Animations** | `@keyframes`, loop infinito or on-trigger | Hardware-accelerated | Loading spinners, hero animations, entrance effects |
| **Micro-interactions** | Feedback visual pequeno (button press, form validation) | Performance-optimized | User feedback, engagement |
| **Cubic Bezier Easing** | Custom easing functions for natural motion | Smooth, professional feel | `cubic-bezier(0.34, 1.56, 0.64, 1)` |
| **Parallax Effects** | Background move slower than foreground | Use sparingly, can impact performance | Hero sections, depth creation |
| **Page Transitions** | Fade/slide entre páginas | JavaScript frameworks (Next.js, Framer Motion) | SPA navigation |
| **GPU-Accelerated Only** | Animate APENAS `opacity` e `transform` | 60fps rendering | Critical for performance |

### G. CODE ORGANIZATION BEST PRACTICES

| Aspecto | Implementação | Benefício |

### 3. JAVASCRIPT MODERNO - ESSENCIAL

```javascript
// 1. Intersection Observer para Lazy Loading
const images = document.querySelectorAll('img[data-src]');
const imageObserver = new IntersectionObserver((entries, observer) => {
    entries.forEach(entry => {
        if (entry.isIntersecting) {
            const img = entry.target;
            img.src = img.dataset.src;
            observer.unobserve(img);
        }
    });
});
images.forEach(img => imageObserver.observe(img));

// 2. Smooth Scroll Behavior
document.querySelectorAll('a[href^="#"]').forEach(anchor => {
    anchor.addEventListener('click', e => {
        e.preventDefault();
        const target = document.querySelector(anchor.getAttribute('href'));
        target?.scrollIntoView({ behavior: 'smooth' });
    });
});

// 3. Accessibility: Focus Management
let allFocusableElements = document.querySelectorAll(
    'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
);

// 4. Performance: Debounced Resize Handler
function debounce(func, delay) {
    let timeoutId;
    return function(...args) {
        clearTimeout(timeoutId);
        timeoutId = setTimeout(() => func(...args), delay);
    };
}

window.addEventListener('resize', debounce(() => {
    console.log('Resize event fired');
}, 250));

// 5. Form Validation
const form = document.querySelector('form');
form?.addEventListener('submit', e => {
    if (!form.checkValidity()) {
        e.preventDefault();
        // Show validation messages
    }
});
```
