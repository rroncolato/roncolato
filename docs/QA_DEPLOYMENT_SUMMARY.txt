CRITICAL SECURITY FIXES REQUIRED
================================================================================

BEFORE PRODUCTION DEPLOYMENT:

1. GENERATE NEW JWT_SECRET
   Command: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   Output: 64-character hex string (e.g., a1b2c3d4...)
   Where to set:
     - .env.production: JWT_SECRET=<generated>
     - Vercel dashboard: Settings → Environment Variables
   Why: Default secret is publicly known, would allow token forgery

2. CHANGE ADMIN CREDENTIALS
   Old: admin / senha123
   New:
     - Username: [unique, not "admin"]
     - Password: [20+ chars, mixed case, numbers, symbols]
   Where to set:
     - .env.production: ADMIN_USER=<new>, ADMIN_PASS=<new>
     - Vercel dashboard: Settings → Environment Variables
   Why: Default credentials are easily guessable

3. CONFIGURE PRODUCTION reCAPTCHA KEYS
   Steps:
     1. Go to https://www.google.com/recaptcha/admin
     2. Create new site for rroncolato.com.br
     3. Select reCAPTCHA v3
     4. Copy Site Key and Secret Key
     5. Update /src/admin/login.html line 138 (Site Key)
     6. Update .env.production (Secret Key)
     7. Update Vercel environment variables
   Why: Test keys don't provide real bot protection
